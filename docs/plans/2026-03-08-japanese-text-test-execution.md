# Mobile Inbox 日本語テスト 実行タスク定義書

**作成日**: 2026-03-08
**担当**: Planning Team (Clio)
**タスクID**: jp_test
**ステータス**: In Progress
**対象**: Mobile Inbox の日本語表示・入力に関する包括的テスト

---

## 1. テスト概要

Mobile Inboxにおける日本語文字列の表示、入力、処理に関する包括的なテストを実施する。

**対象範囲**:
- 文字エンコーディング（UTF-8）
- 日本語フォントレンダリング（Noto Sans JP、M PLUS）
- 特殊文字（サロゲートペア、異体字、IVS、絵文字）
- 文字化け対応

---

## 2. 背景と目的

### 2.1 背景

Claw Empireは日本語ユーザー（manabe）を中心に展開されており、Mobile Inboxでは日本語の正確な表示と入力が必須要件となる。現状、以下の懸念が存在する：

- 文字エンコーディングの不整合による文字化け
- 日本語フォントのデバイス別レンダリング差異
- サロゲートペア・異体字・IVSの正確な表示
- 絵文字と日本語の混在表示

### 2.2 目標

| 目標 | 成功基準 |
|:-----|:---------|
| **UTF-8対応** | 全テスト文字列が正しくエンコード・デコードされる |
| **フォントレンダリング** | 各デバイスで文字が正しく表示される |
| **特殊文字対応** | サロゲートペア、異体字、IVSが正しく表示される |
| **文字化け対策** | エラー時の代替表示（トースト・豆腐文字）が機能する |

## 4. 受入基準（Acceptance Criteria）

### 4.1 P0 (Critical) - ブロッカー

| ID | テスト項目 | 合格条件 | 失敗条件 |
|:---|:-----------|:---------|:---------|
| **JP-P0-001** | UTF-8エンコーディング | 入力・出力共にUTF-8 | 文字化け発生 |
| **JP-P0-002** | ひらがな表示 | 全ひらがなが正しく表示される | 代替文字表示 |
| **JP-P0-003** | カタカナ表示 | 全カタカナが正しく表示される | 代替文字表示 |
| **JP-P0-004** | 漢字表示（常用漢字） | 常用漢字2136字が正しく表示される | 代替文字表示 |
| **JP-P0-005** | サロゲートペア | 𠮷、𠮷などが正しく表示される | 文字化けまたは分割表示 |

### 4.2 P1 (High) - 機能要件

| ID | テスト項目 | 合格条件 | 失敗条件 |
|:---|:-----------|:---------|:---------|
| **JP-P1-001** | 異体字セレクタ（IVS） | 異体字シーケンスが正しく表示される | 標準字体のみ表示 |
| **JP-P1-002** | 絵文字と日本語混在 | 絵文字+日本語が正しく表示される | 絵文字または文字が崩れる |
| **JP-P1-003** | 全角・半角混在 | 全角英数と半角英数が正しく表示される | 幅が不正 |
| **JP-P1-004** | 入力フィールド（IME） | 日本語入力が正しく処理される | 入力文字が消える |
| **JP-P1-005** | 文字数カウント | サロゲートペアを1文字としてカウント | 2文字としてカウント |

### 4.3 P2 (Medium) - 運用改善

| ID | テスト項目 | 合格条件 | 失敗条件 |
|:---|:-----------|:---------|:---------|
| **JP-P2-001** | フォントフォールバック | サポート外文字で代替フォント適用 | 崩れた表示 |
| **JP-P2-002** | 文字化けトースト通知 | エラー時に適切な通知が表示される | 通知未送信 |
| **JP-P2-003** | CI/CD UTF-8環境 | 全環境でUTF-8が保証される | エンコーディングエラー |
| **JP-P2-004** | レスポンスヘッダー | `charset=utf-8` が含まれる | 含まれない |
| **JP-P2-005** | デバイス別レポート | 各デバイスのレンダリング結果 documented | 未記録 |

#### JP-002: サロゲートペア

```json
{
  "category": "surrogate_pairs",
  "strings": [
    "𠮷野屋",        // つちよし（𠮷はサロゲートペア）
    "𠮷田",          // つちよしの「𠮷」
    "𩸽定食",        // ほっけ（魚へんに強）
    "髙橋",          // はしごだか（異体字）
    "﨑",            // たつさき（立つ崎）
    "𠮷𠮷𠮷",        // サロゲートペア連続
    "葛飾区",        // かつしか（くつの文字が旧字体）
    "神羽",          // しんさき（⻌が異なる）
    "徳島",          // 徳の異体字
    "澁谷"           // 渋の異体字（三点水）
  ]
}
```

#### JP-003: 異体字セレクタ（IVS）

```json
{
  "category": "ivs",
  "strings": [
    "葛󠄀区",        // IVSによる葛の旧字体
    "葛󠄁区",        // IVSバリエーション
    "吉󠄀",          // IVSによる「つちよし」
    "神󠄀",          // IVSによる神の異体字
    "辺󠄀",          // IVSによる辺の異体字
    "𠮷󠄀",          // サロゲートペア + IVS
    "⻯󠄀",          // 部首変化
    "﨔󠄀"           // 辶の異体字 + IVS
  ]
}
```

#### JP-004: 絵文字と日本語混在

```json
{
  "category": "emoji_japanese",
```

#### JP-005: 全角・半角混在

```json
{
  "category": "full_half_width",
  "strings": [
    "ＡＢＣ１２３abc123",      // 全角英数 + 半角英数
    "こんにちはHello世界",    // ひらがな + 英字 + 漢字
    "2026年3月8日",            // 半角数字 + 漢字
    "Ｔｅｓｔテスト",          // 全角英字 + カタカナ
    "ｶﾀｶﾅカタカナ",          // 半角カナ + 全角カナ
    "「引用符」vs\"ダブルクォート\"",
    "（括弧）vs(parenthesis)"
  ]

```

#### JP-006: 境界値テスト

## 7. CI/CD文字エンコーディング監査仕様（インフラセキュリティ）

### 7.1 チェック項目

| 項目 | 設定場所 | 期待値 |
|:-----|:---------|:-------|
| **LANG環境変数** | Dockerfile、GitHub Actions | `C.UTF-8` |
| **LC_ALL環境変数** | Dockerfile、GitHub Actions | `C.UTF-8` |
| **Node.jsエンコーディング** | package.json scripts | `--experimental-loader` UTF-8指定 |
| **Dockerベースイメージ** | Dockerfile | `node:slim`（UTF-8対応） |
| **Nginx charset** | nginx.conf | `utf-8` |

### 7.2 監査スクリプト

```bash
#!/bin/bash
# scripts/audit-encoding.sh

echo "=== UTF-8 Encoding Audit ==="

# 環境変数チェック
echo "LANG: $LANG"
echo "LC_ALL: $LC_ALL"

# Node.js エンコーディングチェック
node -e "console.log('Node encoding:', process.env.LANG || 'default')"

# ソースファイルエンコーディングチェック
find src server -type f -name "*.ts" -o -name "*.tsx" | while read file; do
  encoding=$(file -b --mime-encoding "$file")
  if [ "$encoding" != "utf-8" ]; then
    echo "WARNING: $file is $encoding (expected utf-8)"
  fi
done

# Dockerfile チェック
grep -i "FROM.*node" Dockerfile | head -1
grep -i "ENV.*LANG\|ENV.*LC_ALL" Dockerfile || echo "WARNING: No LANG/LC_ALL in Dockerfile"

echo "=== Audit Complete ==="
```

## 12. リスクと対応策

| リスク | 影響 | 対応策 |
|:-------|:-----|:-------|
| IVS表示非対応デバイス | 中 | 代替文字表示、フォールバック処理 |
| サロゲートペア文字化け | 高 | エンコーディング統一、テスト網羅 |

| フォントライセンス | 低 | Noto Sans JP（OFL）、M PLUS（SIL）使用 |
